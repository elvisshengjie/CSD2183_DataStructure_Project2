APSC Overview

The Area-Preserving Segment Collapse (APSC) algorithm simplifies a polygon boundary by repeatedly collapsing 4-vertex windows A→B→C→D into A→E→D, where E is placed on a line parallel to AD that preserves the enclosed area exactly.

The key insight is that E must lie on the line defined by: a·xₑ + b·yₑ + c = 0 where a = y_D − y_A, b = x_A − x_D, and c is derived from the four vertex coordinates.

Among all points on this line, the optimal E minimizes areal displacement. It is placed at the intersection of the area-preserving line with either AB or CD, depending on the topological configuration of B and C relative to AD.

A topology check prevents self-intersections: before applying each collapse, we verify that neither new edge AE nor ED intersects any existing ring segment.

Data Structures Used
Ring (doubly-linked list)
O(1) vertex removal and neighbor lookup after collapse
Min-Heap (priority queue)
Always picks cheapest collapse. O(log n) push/pop
Generation counter
Lazy invalidation — stale candidates are skipped in O(1)
Intersection check
AE and ED tested against all non-adjacent ring segments
Overall complexity: O(n log n)
Per collapse update: O(k) where k = segments tested
Priority queue operations: O(log n) amortized
Placement Function (from paper)
function placement(A, B, C, D):
Compute E↔ line: a=yD-yA, b=xA-xD, c=-yB·xA+(yA-yC)·xB+(yB-yD)·xC+yC·xD
if side(B,AD) == side(C,AD): # same side of AD
if dist(B,AD) > dist(C,AD): return intersect(E↔, AB)
else: return intersect(E↔, CD)
else: # opposite sides of AD
if side(B,AD) == side(E↔,AD): return intersect(E↔, AB)
else: return intersect(E↔, CD)
